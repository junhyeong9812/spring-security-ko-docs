# 04. 메타데이터 인코딩 (Metadata Encoding)

## 무엇을 / 왜

지금까지(02·03챕터)는 **서버가 받은 자격증명을 푸는** 쪽이었다. 이 챕터는 그 반대편, 즉 **클라이언트가 자격증명을 payload 메타데이터에 싣는** 인코더와, 그 표현을 담는 값 객체를 다룬다. (`rsocket/src/main/java/org/springframework/security/rsocket/metadata/`)

RSocket에서 인증 정보는 데이터 본문이 아니라 메타데이터에 실린다. Spring의 `RSocketRequester`는 메타데이터 인코딩을 `Encoder`에 위임하므로, 이 모듈은 자격증명용 `Encoder` 구현을 제공해 클라이언트가 표준 포맷으로 자격증명을 보낼 수 있게 한다.

## 핵심 타입

```
값 객체 (자격증명 표현)
  ├─ UsernamePasswordMetadata   getUsername()/getPassword()
  └─ BearerTokenMetadata        getToken()

인코더 (값 객체 → DataBuffer, 서버로 전송)
  ├─ SimpleAuthenticationEncoder        UsernamePasswordMetadata → Simple (표준, 5.3)
  ├─ BearerTokenAuthenticationEncoder   BearerTokenMetadata → Bearer (표준, 5.3)
  ├─ BasicAuthenticationEncoder         UsernamePasswordMetadata → Basic (@Deprecated)
  └─ BasicAuthenticationDecoder         서버측 디코더 (레거시 Basic)
```

모든 인코더는 Spring의 `AbstractEncoder<T>`를 상속하므로, `RSocketStrategies`에 등록하면 `RSocketRequester`가 메타데이터를 인코딩할 때 자동으로 쓰인다.

## 표준 인코딩: Simple과 Bearer

표준 인코더 둘은 모두 MIME 타입 `message/x.rsocket.authentication.v0`를 쓰고, 실제 바이트 인코딩을 RSocket 라이브러리의 `AuthMetadataCodec`에 위임한다. 02챕터의 `AuthenticationPayloadExchangeConverter`가 바로 이 포맷을 푼다 — 인코딩(클라이언트)과 디코딩(서버)이 같은 코덱으로 짝을 이룬다.

```
[클라이언트]                                  [서버]
UsernamePasswordMetadata("user","pass")
   │ SimpleAuthenticationEncoder
   │ AuthMetadataCodec.encodeSimpleMetadata
   ▼
message/x.rsocket.authentication.v0 바이트 ──전송──▶ payload 메타데이터
                                                      │ AuthenticationPayloadExchangeConverter
                                                      │ AuthMetadataCodec.readWellKnownAuthType=SIMPLE
                                                      │ readUsername/readPassword
                                                      ▼
                                       UsernamePasswordAuthenticationToken.unauthenticated
```

`SimpleAuthenticationEncoder.encodeValue()`의 핵심은 한 줄이다.

```java
ByteBuf simpleAuthentication = AuthMetadataCodec.encodeSimpleMetadata(
        allocator, username.toCharArray(), password.toCharArray());
return factory.wrap(simpleAuthentication);
```

`BearerTokenAuthenticationEncoder`도 동일한 구조로 `encodeBearerMetadata(allocator, token.toCharArray())`를 호출한다. 둘 다 Netty `ByteBuf`를 만들어 `NettyDataBufferFactory`로 감싸 반환한다(버퍼 팩토리가 Netty가 아니면 내부 기본 팩토리 사용).

## 레거시: Basic 인코딩

`BasicAuthenticationEncoder`는 표준화 이전 포맷이라 `@Deprecated`다. `AuthMetadataCodec`을 쓰지 않고 직접 바이트를 조립한다.

```java
// ${username-bytes-length}(4바이트) ${username-bytes} ${password-bytes}
metadata.write(usernameBytesLengthBytes);  // username 길이 (int, 4바이트)
metadata.write(usernameBytes);             // username UTF-8
metadata.write(password.getBytes(UTF_8));  // password UTF-8
```

서버측 `BasicAuthenticationDecoder`가 이 길이-접두 포맷을 역으로 읽어 `UsernamePasswordMetadata`를 복원하고, `BasicAuthenticationPayloadExchangeConverter`(02챕터, deprecated)가 그것을 `Authentication`으로 바꾼다. `UsernamePasswordMetadata.BASIC_AUTHENTICATION_MIME_TYPE`(`message/x.rsocket.authentication.basic.v0`)이 이 레거시 경로의 MIME 타입이다. 신규 코드는 표준 Simple로 옮겨야 한다.

## 인코더와 디코더의 짝

이 모듈의 메타데이터 계층은 클라이언트-서버 양쪽 절반을 모두 제공한다. 짝을 정리하면 다음과 같다.

```
포맷      클라이언트(인코더)                서버(디코더/컨버터)                        상태
──────────────────────────────────────────────────────────────────────────────────
Simple   SimpleAuthenticationEncoder      AuthenticationPayloadExchangeConverter   표준
Bearer   BearerTokenAuthenticationEncoder AuthenticationPayloadExchangeConverter   표준
Basic    BasicAuthenticationEncoder       BasicAuthenticationDecoder +             deprecated
                                          BasicAuthenticationPayloadExchangeConverter
```

표준 컨버터(`AuthenticationPayloadExchangeConverter`)는 같은 MIME 타입 안에서 SIMPLE/BEARER를 코덱이 읽은 `WellKnownAuthType`으로 구분하므로, 서버는 컨버터 하나로 두 인코더의 출력을 모두 처리한다.

## 설계 포인트 / 확장점

- **Spring Encoder 추상화 재사용**: `AbstractEncoder`를 상속해 `RSocketStrategies`/`RSocketRequester`의 메타데이터 인코딩 파이프라인에 자연스럽게 끼워진다. 별도 클라이언트 API가 필요 없다.
- **코덱 위임**: 표준 인코더는 바이트 레이아웃을 직접 다루지 않고 RSocket의 `AuthMetadataCodec`에 맡겨, 인코딩/디코딩 양쪽이 프로토콜 명세와 어긋나지 않는다.
- **레거시 격리**: Basic 경로는 전용 값 객체·MIME 타입·인코더/디코더로 분리돼 `@Deprecated`로 명확히 표시된다. 표준 경로와 섞이지 않는다.
- **값 객체 분리**: `UsernamePasswordMetadata`/`BearerTokenMetadata`는 인코딩 방식과 무관한 순수 자격증명 표현이라, 같은 값으로 Simple/Basic 어느 인코더든 쓸 수 있다.

## 정리

메타데이터 계층은 클라이언트가 자격증명을 payload 메타데이터로 싣는 인코더와 그 값 객체를 제공한다. 표준 `SimpleAuthenticationEncoder`/`BearerTokenAuthenticationEncoder`는 RSocket의 `AuthMetadataCodec`으로 `message/x.rsocket.authentication.v0` 포맷을 만들고, 서버측 `AuthenticationPayloadExchangeConverter`가 같은 코덱으로 이를 푼다. 레거시 Basic 경로는 deprecated로 분리돼 있다. 이로써 02챕터의 서버측 인증 추출과 본 챕터의 클라이언트측 인코딩이 한 쌍으로 맞물리며 RSocket 보안의 자격증명 전송 경로가 완성된다.
